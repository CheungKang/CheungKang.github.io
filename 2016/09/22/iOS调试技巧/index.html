###NSLog打印

最基础的调试方法, 打印日志. 通过打印出的日志来查看程序运行的路径. 系统NSLog打印的信息较少, 效率较低, 常常使用宏来优化系统的打印方法.

swift版
func CKLog<T>(message: T, file: String = #file, method: String = #function, line: Int = #line) {
#if DEBUG
print("<\((file as NSString).lastPathComponent) : \(line)>, \(method)  \(message)")
#endif
}
OC版
#ifdef DEBUG
#define CKLog(fmt, ...) NSLog((@"<%s : %d> %s  " fmt), [[[NSString stringWithUTF8String:__FILE__] lastPathComponent]   UTF8String], __LINE__, __PRETTY_FUNCTION__,  ##__VA_ARGS__);
#else
#define CKLog(...)
#endif

###断点

  • 普通断点

断点(Breakpoint):当程序运行到断点处时会暂停运行.

  • 条件断点

条件断点是在普通断点的基础上添加了判断条件。当程序执行到断点处，并且满足设置的条件时断点才会有效.

在Edit Edit Breakpoint…窗口中有下面四个输入项：

1. Condition 设置的条件。
2. Ignore 表示忽略多少次之后断点有效。例如：不设置条件，Ignore设置为4，则i=4时程序在断点处停止。
3. Action 在程序断点处执行的操作（执行完这个操作后代码停止运行，此时设置断点处的代码还没有执行）。这里的操作是LLDB语句，关于LLDB会在下面介绍。
4. Options 当选中时，执行完Action的操作后代码不会停止，就像没有设置断点一样。
  • 异常断点Exception BreakPoint

在设置异常断点的情况下 当程序crash时 Xcode会帮我们定位到crash产生的位置。

  • 符号断点Symbolic Breakpoint

符号断点能够为某一个方法或者 某一个类的某一个方法设置断点。
